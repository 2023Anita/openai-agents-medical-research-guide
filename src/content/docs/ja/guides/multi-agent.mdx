## 一言で説明

Multi-agent は複雑な作業を専門 agent に分ける設計です。

## たとえ

医学論文は一人で完結しません。文献の先生、統計の先生、執筆の先生が必要になることがあります。

## 最小コード

```python
from agents import Agent, Runner

literature_agent = Agent(name="Literature Assistant", instructions="Only provide search strategy and evidence notes.")
statistics_agent = Agent(name="Statistics Assistant", instructions="Only provide study design and statistical advice.")

orchestrator = Agent(
    name="Medical Research Orchestrator",
    instructions="Call specialist tools and integrate a concise research draft.",
    tools=[
        literature_agent.as_tool(
            tool_name="literature_helper",
            tool_description="Provide literature search strategy and evidence signals.",
        ),
        statistics_agent.as_tool(
            tool_name="statistics_helper",
            tool_description="Provide study design and statistical analysis suggestions.",
        ),
    ],
)

result = Runner.run_sync(orchestrator, "Design a retrospective ICU sepsis study.")
print(result.final_output)
```

## 医療研究での使い方

最終回答は主 agent が担当し、専門 agent は tool として呼ぶのが扱いやすいです。安全境界、出力形式、語調を統一できます。

## よくある失敗

- すべての専門 agent が万能助手になってしまう。
- agent が多すぎてコストとデバッグ負荷が増える。
- 統合役がなく、断片的な回答になる。

## 練習

プライバシー、匿名化、倫理審査だけを担当する `ethics_agent` を追加してください。
