## 一言で説明

Agents SDK は、ツールを使い、複数ステップを実行し、専門 agent と協力し、デバッグできる AI ワークフローを作るための Python ツールキットです。

一度だけ答えればよい場合は Responses API で十分なことが多いです。関数を呼ぶタイミング、統計アシスタントを呼ぶタイミング、研究プロジェクトの文脈を保存するタイミング、安全境界を発火するタイミングをモデルに判断させたいなら、Agents SDK が向いています。

## 医療研究での用途

向いていること：

- 文献課題の整理：研究したい関係を PICO、キーワード、採用基準へ分解する。
- 統計計画の草案：研究デザイン、変数、アウトカム、交絡因子、欠測処理を提案する。
- 論文支援：Introduction や Methods の構造化草案を作る。
- ローカル研究コパイロット：同じ研究課題を複数回やり取りしながら、人間の確認を前提に進める。

向いていないこと：

- 患者個別の診断。
- 治療や投薬の助言。
- トリアージや救急判断。
- 検証されていない自動研究結論。

## 全体マップ

| 概念 | やさしい意味 | 研究でのたとえ | よく使うコード |
| --- | --- | --- | --- |
| `Agent` | 役割、規則、ツールを持つモデル | 文献、統計、執筆助手 | `Agent(...)` |
| `Runner` | agent を実行する | プロジェクト管理者 | `Runner.run(...)` |
| `Tools` | agent が呼べる能力 | 検索、尺度説明、整形 | `@function_tool` |
| `Structured Output` | 固定フィールドの出力 | 研究課題、方法、リスク | `output_type=BaseModel` |
| `Sessions` | 会話の記憶 | 同じ研究課題の文脈 | `SQLiteSession(...)` |
| `Multi-agent` | 専門家の協力 | 文献 + 統計 + 執筆 | `Agent.as_tool()` |
| `Guardrails` | 安全チェック | 臨床助言を止める | `input_guardrail` |
| `Tracing` | 実行記録 | ツールとエラーを見る | `trace(...)` |

## 練習

単純な文献要約なら Responses API と Agents SDK のどちらを使うか。データ辞書を読み、ツールを呼び、計画を作り、文脈を保存するならどちらを使うか説明してください。
