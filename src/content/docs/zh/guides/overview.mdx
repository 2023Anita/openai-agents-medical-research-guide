## 一句话解释

Agents SDK 是一个 Python 工具包，用来构建“能调用工具、能多步执行、能协作、能被调试”的 AI 工作流。

如果你只是问模型一句话，Responses API 通常就够了。如果你要让模型决定什么时候调用函数、什么时候让统计助手介入、什么时候保存课题上下文、什么时候触发安全边界，就需要 Agents SDK。

## 医疗科研场景

它适合做：

- 文献问题拆解：把“我想研究 X 和 Y 的关系”变成 PICO、关键词、纳排标准。
- 统计方案初稿：给出研究设计、变量、结局、混杂因素、缺失值处理建议。
- 论文草案辅助：生成 Introduction/Methods 的结构化初稿。
- 本地科研副驾驶：围绕同一个研究项目多轮迭代，但始终要求人工复核。

它不适合直接做：

- 患者个体诊断。
- 治疗方案或用药建议。
- 分诊、急救、医学责任判断。
- 未经验证的自动统计结论。

## 一页总览

| 概念 | 白话解释 | 医疗科研类比 | 常见代码 |
| --- | --- | --- | --- |
| `Agent` | 带角色、规则、工具的模型 | 文献助手、统计助手、写作助手 | `Agent(...)` |
| `Runner` | 把 agent 跑起来 | 项目经理推进任务 | `Runner.run(...)` |
| `Tools` | agent 可调用的能力 | 检索、量表解释、格式化摘要 | `@function_tool` |
| `Structured Output` | 固定输出字段 | 研究问题、方法、风险提示 | `output_type=BaseModel` |
| `Sessions` | 多轮记忆 | 记住同一个课题 | `SQLiteSession(...)` |
| `Multi-agent` | 多专家协作 | 文献、统计、写作分工 | `Agent.as_tool()` |
| `Guardrails` | 安全边界 | 拦住诊疗建议 | `input_guardrail` |
| `Tracing` | 执行记录 | 查看工具调用和错误 | `trace(...)` |
