# Portfolio Case Study

## From slides to GitHub-native interactive AI course labs

This project is a portfolio case for a new teaching format: instead of presenting the OpenAI Agents SDK as a slide deck, the course becomes a working repository that learners can read, answer, run, fork, remix, and publish.

Live case page: <https://2023Anita.github.io/openai-agents-medical-research-guide/case-study/>

## The problem

Fast-moving AI frameworks are hard to teach with passive slides. Slides can explain vocabulary, but they rarely leave learners with something they can inspect, run, and turn into their own public project.

The project reframes the teaching unit as a product loop:

```text
Read the concept
-> answer a checkpoint
-> inspect runnable code
-> fork the repository
-> replace the scenario
-> publish a new course lab
```

## The product idea

The course teaches the OpenAI Agents SDK through one practical project: a safety-first medical research assistant. That single spine makes abstract SDK concepts concrete:

- `Agent` and `Runner` become the main assistant workflow.
- `Tools` become local research-support functions.
- `Structured Output` becomes a consistent research brief.
- `Sessions` become project context memory.
- `Multi-agent` becomes literature, statistics, and writing specialists.
- `Guardrails` become the medical safety boundary.
- `Tracing` becomes the debugging and audit trail.

## What makes it portfolio-grade

| Layer | Evidence |
| --- | --- |
| Product | A clear position: not a slide deck, but a GitHub-native interactive course lab. |
| Engineering | Static-first implementation, GitHub Pages deployment, no backend, no login, no API key required for exercises. |
| Education | Repeated chapter rhythm, embedded checkpoints, open practice tasks, and a fork/remix workflow. |
| Domain | Medical research safety boundary appears in the site, README, demo code, contribution guide, and docs. |
| Portfolio | README, screenshots, design spec, case page, demo code, and remix guide are all visible from the repository front door. |

## Safety boundary

This is a medical research education project. It does not provide diagnosis, treatment, triage, medication advice, or patient-specific recommendations. Outputs must be reviewed by human researchers, statisticians, ethics reviewers, or clinical experts.

## Why it matters

The strongest outcome is not only that learners understand the Agents SDK. The stronger outcome is that they can fork the course pattern and publish their own applied AI learning product.
